El error es claro y no tiene que ver con los approvals en sí, sino con la conexión a la red Polygon. Mira estos indicadores clave del error:

"reason":"could not detect network"
"code":"NETWORK_ERROR"
"event":"noNetwork"
"chainId":{} ← vacío, nunca se conectó a la red

Tu bot no está llegando a Polygon. Las aprobaciones ni siquiera se intentan ejecutar porque la conexión RPC falla antes.

Causas y soluciones
1. RPC URL incorrecto o faltante
Tu bot necesita un proveedor RPC de Polygon configurado correctamente. ¿Qué URL estás usando? Debe ser algo como:

https://polygon-rpc.com (público, gratuito pero limitado)
https://polygon-mainnet.g.alchemy.com/v2/TU_API_KEY (Alchemy)
https://polygon-mainnet.infura.io/v3/TU_PROJECT_ID (Infura)

Los públicos gratuitos se saturan frecuentemente y pueden fallar con "noNetwork".
2. El provider de ethers.js no se está inicializando correctamente
Tu bot usa ethers.js v5. La inicialización debe verse así:
javascriptconst { ethers } = require("ethers");

// Opción 1: RPC directo
const provider = new ethers.providers.JsonRpcProvider(
  "https://polygon-mainnet.g.alchemy.com/v2/TU_KEY", 
  137  // ← Importante: especificar chainId explícitamente
);

// Opción 2: Si usas StaticJsonRpcProvider (evita la detección automática de red)
const provider = new ethers.providers.StaticJsonRpcProvider(
  "https://polygon-mainnet.g.alchemy.com/v2/TU_KEY",
  { chainId: 137, name: "polygon" }
);

const wallet = new ethers.Wallet("TU_PRIVATE_KEY", provider);
StaticJsonRpcProvider es la opción recomendada porque no intenta detectar la red automáticamente (que es exactamente donde falla tu bot).
3. Si tu bot corre en Replit u otro hosting
Algunos servicios bloquean conexiones salientes a ciertos RPCs. Si estás en Replit, prueba con un RPC de Alchemy o Infura en lugar de los públicos.
